## 1. Internal Go Event Model

The canonical Go struct used across all application layers. All packages (`ingestor`, `normalizer`, `enricher`, `api`) work with this type.

**File**: `api/internal/models/event.go`

```go
package models

import (
    "time"

    "github.com/google/uuid"
)

// EventCategory is the normalized event classification.
type EventCategory string

const (
    CategoryFloods    EventCategory = "floods"
    CategoryWildfires EventCategory = "wildfires"
)

// EventStatus reflects whether the EONET event is still active.
type EventStatus string

const (
    StatusOpen   EventStatus = "open"
    StatusClosed EventStatus = "closed"
)

// Event is the canonical internal event representation.
// All layers (ingestor, normalizer, enricher, API) use this type.
type Event struct {
    ID          uuid.UUID     `json:"id"            db:"id"`
    SourceID    string        `json:"source_id"     db:"source_id"`
    Source      string        `json:"source"        db:"source"`
    Title       string        `json:"title"         db:"title"`
    Category    EventCategory `json:"category"      db:"category"`
    Status      EventStatus   `json:"status"        db:"status"`
    GeomType    *string       `json:"geometry_type" db:"geom_type"`
    Latitude    *float64      `json:"latitude"      db:"latitude"`
    Longitude   *float64      `json:"longitude"     db:"longitude"`
    CountryName *string       `json:"country_name"  db:"country_name"`
    StateName   *string       `json:"state_name"    db:"state_name"`
    EventDate   *time.Time    `json:"event_date"    db:"event_date"`
    SourceURL   *string       `json:"source_url"    db:"source_url"`
    RawPayload  []byte        `json:"-"             db:"raw_payload"`
    IngestedAt  time.Time     `json:"ingested_at"   db:"ingested_at"`
    EnrichedAt  *time.Time    `json:"enriched_at"   db:"enriched_at"`
}
```

### Field Notes

| Field        | Type          | Notes                                                                      |
|--------------|---------------|----------------------------------------------------------------------------|
| `ID`         | uuid.UUID     | Generated by PostgreSQL `gen_random_uuid()`. Never set in application code. |
| `SourceID`   | string        | Original EONET event ID. Unique. Used for upsert deduplication.            |
| `Source`     | string        | Always `"eonet"` in MVP. Field supports future data source extensions.     |
| `GeomType`   | *string       | Denormalised from PostGIS geometry for fast API serialization.              |
| `Latitude`   | *float64      | Centroid lat for Point events. Null for Polygon or missing geometry.        |
| `Longitude`  | *float64      | Centroid lon for Point events. Null for Polygon or missing geometry.        |
| `CountryName`| *string       | Null until enrichment runs (F-005). Populated by PostGIS ST_Intersects.    |
| `StateName`  | *string       | Null until enrichment runs. Null if event falls outside Nigeria.            |
| `RawPayload` | []byte (JSONB)| Complete EONET response. Never serialised to JSON API responses (`json:"-"`)|
| `EnrichedAt` | *time.Time    | Null until enrichment runs. Used to target un-enriched events.              |

## 2. Database Schema: events table

**Migration file**: `api/db/migrations/001_create_events.sql`

```sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS postgis;

CREATE TABLE events (
    id            UUID             PRIMARY KEY DEFAULT gen_random_uuid(),
    source_id     TEXT             NOT NULL UNIQUE,
    source        TEXT             NOT NULL DEFAULT 'eonet',
    title         TEXT             NOT NULL,
    category      TEXT             NOT NULL CHECK (category IN ('floods', 'wildfires')),
    status        TEXT             NOT NULL DEFAULT 'open'
                                   CHECK (status IN ('open', 'closed')),
    geom          geometry(Geometry, 4326),
    geom_type     TEXT,
    latitude      DOUBLE PRECISION,
    longitude     DOUBLE PRECISION,
    country_name  TEXT,
    state_name    TEXT,
    event_date    TIMESTAMPTZ,
    source_url    TEXT,
    raw_payload   JSONB,
    ingested_at   TIMESTAMPTZ      NOT NULL DEFAULT now(),
    enriched_at   TIMESTAMPTZ
);

-- Performance indexes
CREATE INDEX idx_events_category    ON events(category);
CREATE INDEX idx_events_status      ON events(status);
CREATE INDEX idx_events_state_name  ON events(state_name);
CREATE INDEX idx_events_event_date  ON events(event_date DESC);
CREATE INDEX idx_events_geom        ON events USING GIST(geom);
```

### Column Definitions

| Column          | Type                     | Constraints                  | Notes                                                      |
|-----------------|--------------------------|------------------------------|------------------------------------------------------------|
| `id`            | UUID                     | PRIMARY KEY, default gen_random_uuid() | Internal identifier. Exposed in API as `id`. |
| `source_id`     | TEXT                     | UNIQUE NOT NULL              | EONET event ID. Unique constraint enables idempotent upserts. |
| `source`        | TEXT                     | NOT NULL, default `'eonet'`  | Extensible for future data sources.                         |
| `category`      | TEXT                     | CHECK IN ('floods','wildfires')| CHECK constraint enforces MVP event type scope.            |
| `status`        | TEXT                     | CHECK IN ('open','closed')   | `open` = active event; `closed` = resolved/cleared.        |
| `geom`          | geometry(Geometry, 4326) | nullable                     | WGS84. Accepts both Point and Polygon from EONET.          |
| `geom_type`     | TEXT                     | nullable                     | Denormalised: `'Point'` or `'Polygon'`. Skips PostGIS parse in API. |
| `latitude`      | DOUBLE PRECISION         | nullable                     | Centroid latitude for Point events. Null for Polygon.       |
| `longitude`     | DOUBLE PRECISION         | nullable                     | Centroid longitude for Point events. Null for Polygon.      |
| `country_name`  | TEXT                     | nullable                     | Null until F-005 enrichment runs.                          |
| `state_name`    | TEXT                     | nullable                     | Null until F-005 enrichment runs. Null if outside Nigeria. |
| `event_date`    | TIMESTAMPTZ              | nullable                     | Original event date from EONET. Null if EONET omits it.    |
| `source_url`    | TEXT                     | nullable                     | Link to the original EONET event page.                     |
| `raw_payload`   | JSONB                    | nullable                     | Complete EONET API payload. For debugging and re-processing.|
| `ingested_at`   | TIMESTAMPTZ              | NOT NULL, default now()      | Set at insertion time by the database.                     |
| `enriched_at`   | TIMESTAMPTZ              | nullable                     | Set when PostGIS enrichment completes. `null` = not yet enriched. |

### Upsert Strategy (Deduplication)

The application uses PostgreSQL `INSERT ... ON CONFLICT` for idempotent ingestion:

```sql
INSERT INTO events (source_id, source, title, category, status, geom, geom_type,
                    latitude, longitude, event_date, source_url, raw_payload)
VALUES ($1, $2, $3, $4, $5, ST_GeomFromGeoJSON($6), $7, $8, $9, $10, $11, $12)
ON CONFLICT (source_id)
DO UPDATE SET
    status      = EXCLUDED.status,
    title       = EXCLUDED.title,
    raw_payload = EXCLUDED.raw_payload;
```

This ensures:
- First insert: full event created
- Re-ingestion: `status` updated (handles open → closed transitions), `title` refreshed if changed
- Fields like `country_name`, `state_name`, `enriched_at` are NOT overwritten on re-ingestion

## 3. Database Schema: admin_boundaries table

**Migration file**: `api/db/migrations/002_create_admin_boundaries.sql`

```sql
CREATE TABLE admin_boundaries (
    id            SERIAL           PRIMARY KEY,
    country_code  TEXT             NOT NULL,
    country_name  TEXT             NOT NULL,
    adm_level     INTEGER          NOT NULL CHECK (adm_level IN (0, 1, 2)),
    adm_name      TEXT             NOT NULL,
    geom          geometry(MultiPolygon, 4326) NOT NULL
);

-- Performance indexes
CREATE INDEX idx_boundaries_country    ON admin_boundaries(country_code);
CREATE INDEX idx_boundaries_adm_level  ON admin_boundaries(adm_level);
CREATE INDEX idx_boundaries_adm_name   ON admin_boundaries(adm_name);
CREATE INDEX idx_boundaries_geom       ON admin_boundaries USING GIST(geom);
```

### Column Definitions

| Column         | Type                        | Notes                                                               |
|----------------|-----------------------------|---------------------------------------------------------------------|
| `id`           | SERIAL                      | Internal primary key. Not exposed in API.                           |
| `country_code` | TEXT                        | ISO 3166-1 alpha-2 (e.g., `'NG'` for Nigeria).                    |
| `country_name` | TEXT                        | English country name (e.g., `'Nigeria'`).                          |
| `adm_level`    | INTEGER                     | 0 = Country (ADM0), 1 = State/Province (ADM1), 2 = LGA (deferred) |
| `adm_name`     | TEXT                        | English administrative unit name (e.g., `'Benue'`, `'Lagos'`).    |
| `geom`         | geometry(MultiPolygon, 4326)| WGS84. MultiPolygon to handle fragmented boundaries.               |

### Data Sources

| Country | ADM Level | Source                            | URL                                              |
|---------|-----------|-----------------------------------|--------------------------------------------------|
| Nigeria | ADM0      | Humanitarian Data Exchange (HDX)  | https://data.humdata.org/dataset/cod-ab-nga      |
| Nigeria | ADM1      | Humanitarian Data Exchange (HDX)  | https://data.humdata.org/dataset/cod-ab-nga      |
| Nigeria | ADM2      | HDX (deferred — not loaded in MVP)| n/a until v0.6+                                  |

**Naming convention**: Use the English administrative names exactly as they appear in the HDX dataset. Do not translate, abbreviate, or normalise state names. This ensures `state_name` in the events table matches `adm_name` in the boundaries table.

## 4. Canonical Enrichment Query

This is the exact SQL used by the enrichment engine (F-005). No deviation permitted without updating this spec.

**Migration or seed file**: `api/db/migrations/003_seed_nigeria_boundaries.sql` (loads the GeoJSON)

**Application query** (run after each ingestion batch):

```sql
UPDATE events e
SET
    country_name = b_country.adm_name,
    state_name   = b_state.adm_name,
    enriched_at  = now()
FROM
    admin_boundaries b_country,
    admin_boundaries b_state
WHERE
    e.geom IS NOT NULL
    AND e.enriched_at IS NULL
    AND b_country.country_code = 'NG'
    AND b_country.adm_level    = 0
    AND b_state.country_code   = 'NG'
    AND b_state.adm_level      = 1
    AND ST_Intersects(e.geom, b_country.geom)
    AND ST_Intersects(e.geom, b_state.geom);
```

**Behaviour**:
- Targets only un-enriched events (`enriched_at IS NULL`) — idempotent
- Events whose geometry falls outside all Nigerian state boundaries get no update (`country_name` and `state_name` remain `null`)
- `enriched_at` is set only on rows that were successfully matched

## 5. Migration Naming Convention

Migrations are numbered sequentially and applied in ascending order at startup.

**Format**: `NNN_description.sql` where `NNN` is zero-padded (001, 002, ...)

```
api/db/migrations/
├── 001_create_events.sql               ← events table
├── 002_create_admin_boundaries.sql     ← admin_boundaries table
└── 003_seed_nigeria_boundaries.sql     ← load Nigeria ADM0 + ADM1 from HDX GeoJSON
```

**Migration runner**: Use `golang-migrate/migrate` with the PostgreSQL driver. Migrations are applied once and tracked in a `schema_migrations` table managed by the tool.

**Rules**:
- Migrations are **append-only**. Never modify a migration that has already been applied.
- Destructive changes (DROP COLUMN, DROP TABLE) require a new numbered migration.
- Data migrations (seeding, backfills) are separate files from schema migrations.

## 6. Local Development Seed Data

**File**: `api/db/seeds/sample_events_nigeria.sql`

A seed dataset for local development, enabling the full application to function without a live EONET connection.

**Requirements for the seed**:
- Minimum 15 events
- Events spread across at least 5 different Nigerian states
- Mix of `floods` and `wildfires` categories
- Mix of `open` and `closed` status
- All events must have valid `geom` coordinates that fall within Nigerian state boundaries
- Some events should be pre-enriched (with `country_name`, `state_name`, `enriched_at` set) and some not (to test enrichment logic)

**Example row**:

```sql
INSERT INTO events (source_id, source, title, category, status, geom, geom_type, latitude, longitude, event_date)
VALUES (
    'EONET_SEED_001',
    'eonet',
    'Flooding along Benue River basin',
    'floods',
    'open',
    ST_SetSRID(ST_MakePoint(8.13, 7.33), 4326),
    'Point',
    7.33,
    8.13,
    '2026-04-01T00:00:00Z'
);
```
